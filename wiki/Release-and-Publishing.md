# Release And Publishing

Use this wiki page as the short version of the release checklist.

## Before Publish

- update `README.md`
- update `docs/`
- update `wiki/`
- make sure the screenshot gallery still reflects the product
- run tests, typecheck, and builds

## Release Steps

1. bump the package version
2. run preflight checks
3. dry-run the package
4. publish to npm
5. cut the GitHub release
6. confirm the public repo surfaces still look polished

For the detailed version, use [../docs/releasing.md](../docs/releasing.md).
