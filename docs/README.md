# Kosmos Docs

This folder is the main documentation hub for Kosmos. Kosmos is positioned as the local trust layer for AI working inside Obsidian-style vaults, repos, prompts, and knowledge bases.

## Recommended Reading Order

1. Start with [Getting Started](getting-started.md).
2. Read [Product Positioning](product-positioning.md) to understand the trust-layer wedge.
3. Keep [CLI And Install Guide](cli-and-install.md) open if you need launch options or `npx` examples.
4. Use [Architecture Guide](architecture.md) to understand what Kosmos is modeling.
5. Learn live workflows in [Live Monitoring Guide](live-monitoring.md).
6. Debug real sessions with [Trace Inspector Guide](trace-inspector.md).
7. Add iteration discipline with [Prompt Versioning Guide](prompt-versioning.md).
8. Use [Troubleshooting](troubleshooting.md) and [FAQ](faq.md) when something feels off.
9. Finish with [Releasing And Publishing](releasing.md) when you are preparing a public release.
