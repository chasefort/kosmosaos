# FAQ

## Does Kosmos require a cloud account?

No. Kosmos is designed to be useful locally.

## Where does it store data?

The primary local database lives under `~/.kosmos/kosmos.db`.

## Is Kosmos only for one framework?

No. It works best when a workspace exposes prompts, tools, agents, files, and runtime events clearly, regardless of the exact stack.

## What kinds of teams is it for?

It is especially useful for:

- local AI products
- agent-heavy prototypes
- prompt-driven internal tools
- teams that want runtime evidence tied directly to files and instructions

## When is Kosmos less useful?

If your project has no prompts, no agent structure, no runtime events, and no file-heavy workflow, the graph will naturally be less interesting.

## Do I have to use live integrations?

No. The static architecture views are still useful on their own. Live integrations make Kosmos much more valuable, but they are not required to get started.
